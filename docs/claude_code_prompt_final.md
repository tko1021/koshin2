# DIPS更新修了者情報CSV作成システム 構築指示（Claude Code用・確定版）

## 目的
登録更新講習機関の修了者マスタ（input/master.xlsx）から、DIPS「更新修了者情報一括登録」
へアップロードするCSVを生成・検証するローカルシステムを構築する。
仕様はDIPS操作マニュアル＜更新講習修了者情報連携編＞Ver2026/3/23 及び国交省ガイドラインに準拠する。

## 入力ファイル（input/ に配置済み）
1. input/master.xlsx … 修了者マスタv3（シート:「修了者マスタ」(ヘッダー3行目/データ4行目以降)、
   「設定」「祝日」「記入要領」。緑色列はExcel数式の自動計算列）
2. input/dips_format.csv … DIPS公式雛形（ヘッダー1行+記載例1行。UTF-8 BOM付き・CRLF）

## 出力CSV仕様（確定。実装前に input/dips_format.csv を読みバイト級で再確認すること）
- UTF-8（雛形にBOM有→出力もBOM付きで一致させる）/ CRLF / カンマ区切り / クォートなし
- 1行目ヘッダーは雛形と完全一致（1文字でも変更するとDIPS側でエラー）
- 1行=修了証明書1件。**1ファイル最大500件**。**同一の技能証明申請者番号は1ファイル内に1件まで**
- ファイル名: 短く保つ（長いとDIPSのエラー結果ファイルが出力不可）。拡張子.csvは変更不可。
  既定: `修了者情報_YYYYMMDD_連番.csv`
- 全17列と設定値（操作マニュアルで確定済み）:

| No | 列 | 設定値 |
|---|---|---|
| 1 | 申請ID | マスタ「申請ID」列（既定=修了証明書番号）。半角英数字のみ |
| 2 | 技能証明申請者番号 | マスタ値。10文字以内 |
| 3 | 登録更新講習機関コード | 設定シートB2 |
| 4 | 登録更新講習機関事務所コード | マスタ「事務所コード」（空なら設定シートB4の既定値） |
| 5 | 区分 | 一等→"1" / 二等→"2"（一等・二等両方保有者は"1"） |
| 6 | 停止処分者向け講習受講有無 | 無→"1"（受講なし）/ 有→"2"（受講あり）※注意: 直感と逆 |
| 7 | 身体適性検査証明書番号 | **固定値 "PA000000000000"**（マスタの内部PA番号は出力しない） |
| 8〜12 | 受検方法/技能証明の条件/検査日/公的な証明書等/備考 | **すべて空欄固定** |
| 13 | 更新講習修了証明書番号 | マスタ値（先頭UC または EL の14桁） |
| 14 | 更新講習修了日 | マスタ「講習修了日」 yyyy/mm/dd |
| 15 | 有効期間満了日 | マスタ値 yyyy/mm/dd（発行日の3か月後の前日） |
| 16 | 状態フラグ | マスタ「登録種別」: 1=新規 / 2=上書き修正 / 3=無効化 |
| 17 | 備考 | 空欄固定（将来用） |

変換表・固定値は config/code_map.yaml、列マッピングは config/mapping.yaml に外出しし、
列はヘッダー名で参照（列順ハードコード禁止）。様式改定時は設定修正のみで追従できる構造とする。

## バリデーション（DIPSサーバ側チェックの完全ローカル再現。出力前に全件実行）
1. 項目数: 出力行のカンマ数=16（17列）
2. 禁則文字: 全項目で「¥ / ; * ? > < _ " | ' ,」を禁止（日付項目の「/」のみ許可）。
   氏名等にカンマ・アンダースコア等が含まれる場合は該当行をエラー表示
3. 必須項目: 申請ID/申請者番号/機関コード/事務所コード/区分/停止処分有無/検査証明書番号/
   修了証明書番号/修了日/有効期間満了日/状態フラグ
4. 形式: 申請ID=半角英数字のみ(空白不可) / 申請者番号≦10文字 / 修了証明書番号=^(UC|EL)\d{12}$ /
   機関コード=^\d{4}$ / 日付=yyyy/mm/dd かつ実在日 / 有効期間満了日は過去日不可(当日比較)
5. 整合: 修了証明書番号内の機関コード=設定値、番号内の発行年月とマスタ発行日の年月一致、
   有効期間満了日=発行日+3か月-1日、申請ID・修了証明書番号の重複（マスタ全行+出力履歴と突合）、
   選択行内の技能証明申請者番号の重複（重複時は自動でファイル分割するか選択解除を促す）
6. 状態フラグ整合: 「2」「3」の行は出力履歴に同一申請IDの「1」登録実績があるか確認し、なければ警告。
   「3」無効化は確認ダイアログを必須とする

## 業務ルール
- DIPS連携期限=修了証明書発行日から5営業日（土日+マスタ「祝日」シート）。一覧で期限超過=赤/残1営業日以内=黄
- 採番はマスタのExcel数式が実施済み。システムは検証と重複チェックのみ（再採番しない）
- Excel読込は文字列欠落防止のため、番号系列は str として扱う（openpyxl data_only=True）

## システム要件
- Python 3.11+ / Streamlit（streamlit run app.py）/ 完全ローカル動作・外部送信なし
- 機能:
  1. マスタ読込・一覧表示（検索/フィルタ: 氏名・修了日・登録種別・登録状況、期限警告色分け）
  2. 出力対象を複数選択 → 一括1ファイル（≦500行）生成。申請者番号重複時は自動分割
  3. 上記バリデーション → エラーは行番号+項目+理由を明示し出力から除外
  4. 出力後セルフテスト: 生成ファイルを再読込し、BOM/CRLF/ヘッダー完全一致/列数17を
     input/dips_format.csv と突合した結果を画面表示
  5. 出力履歴 data/export_log.csv（日時/管理No/氏名/申請ID/修了証明書番号/状態フラグ/ファイル名）。
     同一申請IDの「1：新規」再出力時は警告
  6. アップロード後の運用支援: 「登録済」への更新忘れ一覧、エラー結果ファイル(DIPSが出力)の
     取込・該当行表示機能（任意実装）
- data/, output/, input/ は .gitignore へ（個人情報保護）
- pytest: コード変換(区分・停止処分の逆転含む)/禁則文字/番号形式/日付(3か月後の前日・月末跨ぎ・うるう年)/
  営業日計算/申請者番号重複分割/CSVバイト列(BOM・CRLF・ヘッダー一致)
- README.md: セットアップ、運用フロー（修了→マスタ入力→5営業日内にCSV出力→DIPSアップロード→登録済記録）、
  受講者側の事前準備（DIPS2.0で当機関の事務所コード登録が必要。未登録だとDIPS側エラー）、
  code_map.yaml確認方法、様式改定時の追従手順

## 進め方
1. input/ の2ファイルを読み、本指示のスキーマと相違がないか確認して報告
2. config/mapping.yaml・code_map.yaml を生成し内容を提示
3. 実装 → pytest全件パス → マスタの記入例2行で試験生成し、雛形とのバイト級突合結果を報告
